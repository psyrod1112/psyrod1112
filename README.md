# 2년 포트폴리오 로드맵

## 전체 전략 (왜 이렇게 진행하는가)

프로젝트를 단순히 여러 개 만드는 것이 아니라 다음 흐름을 목표로 한다.

- 난이도 점진 상승  
- 기술 스택 확장  
- 운영 경험 축적  

이 과정을 통해 프로젝트 간 연결된 스토리를 만들고,  
실시간 시스템을 설계하고 운영할 수 있는 개발자로 성장하는 것을 목표로 한다.  
(면접에서 프로젝트의 맥락을 설명하기 위해 매우 중요)

---

# YEAR 1 — 기초 및 실시간 서버 정체성 확립

## 프로젝트 1 — 실시간 멀티플레이 서버 (핵심 프로젝트)

포트폴리오의 중심이 되는 프로젝트.

### 목표

- TCP 또는 WebSocket 기반 통신
- 방 생성 및 매칭 기능
- 상태 동기화 처리
- 채팅 기능 구현

### 필수 요소

- 멀티스레드 또는 비동기 처리 구조
- 서버 권한(authoritative) 구조 설계
- 세션 관리 시스템

### 추가 요소

- 봇 클라이언트를 활용한 동시 접속 테스트
- 패킷 구조 설계 문서 작성

이 프로젝트를 통해  
“실시간 시스템의 동작 원리를 이해하고 설계할 수 있음”을 증명한다.

---

## 인프라 경험 (병행 진행)

실무형 개발자로 성장하기 위해 필수적으로 포함한다.

- 클라우드 배포 (AWS 또는 GCP)
- Docker 컨테이너 환경 구성
- CI/CD 파이프라인 구축 (GitHub Actions)

# YEAR 2 — 백엔드 엔지니어 포지셔닝

## 프로젝트 2 — 확장 가능한 백엔드 시스템

다음 중 하나를 선택하여 구현.

### 옵션 A — 매칭 및 랭킹 서비스

- REST API 서버 구현
- 데이터베이스 설계
- Redis 캐싱 적용

### 옵션 B — 실시간 협업 서버

- 문서 동기화 로직
- 이벤트 기반 아키텍처

### 옵션 C — 게임 로비 플랫폼

- 사용자 인증
- 친구 시스템
- 알림 기능

이 프로젝트의 목표는  
서비스 백엔드를 설계할 수 있는 역량을 보여주는 것이다.

---

## 프로젝트 3 — 운영 경험 중심 프로젝트

프로젝트 규모는 크지 않아도 되며,  
운영 데이터 수집 및 분석 경험 확보를 목표로 한다.

### 목표

- 로그 수집 시스템 구축
- 모니터링 환경 구성
- 성능 측정 및 분석

### 예시 기술

- Prometheus + Grafana
- 로드 테스트 도구 (k6)

이 경험은 면접에서 시스템 운영 이해도를 보여주는 핵심 요소가 된다.
